# Pivot

`F-PIVOT` turns row data into a cross-tab grid while preserving the shared OneGrid row,
column, renderer, accessibility, and server request contracts.

```ts
const pivotModel = {
  rows: ["region", "agency"],
  columns: ["quarter"],
  values: [
    { field: "amount", function: "sum", alias: "amountTotal", label: "Amount" },
    { field: "budget", function: "avg", alias: "avgBudget", label: "Avg budget" }
  ],
  totals: "both",
  subtotals: true
} satisfies PivotModel;

const options: GridOptions<BudgetRow> = {
  columns,
  data,
  rowModel: "client",
  pivot: {
    enabled: true,
    panel: true,
    model: pivotModel
  }
};
```

## Model

- `rows` are the left-side row fields.
- `columns` are the pivoted column dimensions.
- `values` are aggregated measure fields. A string value defaults to `sum`; an object can set
  `function`, `alias`, and a visible `label`.
- `totals: "columns"` adds per-row total columns.
- `totals: "rows"` adds a grand total row.
- `totals: "both"` enables both total directions.
- `subtotals: true` adds subtotal rows for the first row field.

## Client Pivot

Client pivot is calculated in `@onegrid/core` from the filtered and sorted client row set. The DOM
renderer receives generated column groups and pivot rows, then renders them through the normal grid
pipeline. Wrappers do not reimplement pivot behavior.

## Server Pivot

For large datasets use server pivot:

```ts
const options: GridOptions<PivotResultRow> = {
  columns: serverPivotColumns,
  rowModel: "server",
  dataSource,
  pivot: {
    enabled: true,
    serverOnly: true,
    panel: true,
    model: pivotModel
  }
};
```

`DataSource.getRows()` receives `request.pivotModel`. The server owns pivot execution and can return
`GetRowsResult.columns` with the result schema for the current pivot model. OneGrid uses those
returned columns for rendering while the pivot builder still uses the original configured columns as
the field catalog. This keeps 10M to 100M row scenarios out of browser memory without forcing the
browser to infer a server pivot schema.

## Pivot Builder

`pivot.panel: true` renders a keyboard-reachable pivot builder. Users can search available fields,
move fields into row, column, or value buckets, change value aggregation, toggle totals/subtotals,
and apply the draft model. The builder also accepts drag-and-drop between the field catalog and
buckets, shows valid/invalid drop feedback, and displays an insertion line so dragged fields land in
the intended bucket position. Button controls remain available for keyboard and assistive technology
users.

The panel is draft-based. Editing the builder does not mutate the grid until the user chooses
**Apply pivot model**. Apply calls `GridApi.setPivotModel(model)`, which keeps vanilla, React, Vue,
client pivot, and server pivot on the same public contract.

```ts
api.setPivotModel({
  rows: ["region", "agency", "status"],
  columns: ["quarter"],
  values: [{ field: "amount", function: "sum", alias: "amountTotal" }],
  totals: "both",
  subtotals: true
});
```

Client panels state that the browser computed the pivot result. Server panels state that the data
source owns execution and that `pivotModel` was forwarded with the request.
